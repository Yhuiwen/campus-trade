# 功能测试与验收清单

测试前先执行 `sql/init.sql`，启动 MySQL、后端和前端。建议准备 `admin`、`student1`、`student2` 三个浏览器会话。

## 基础检查

- [ ] 首页能显示热门商品、商品列表和分类。
- [ ] 关键字搜索、分类筛选和分页正常。
- [ ] 未登录用户可查看商品详情，但不能发布、下单、收藏或访问个人页面。
- [ ] 错误用户名或密码无法登录。
- [ ] JWT 失效或被篡改后，受保护接口返回 401。
- [ ] 普通用户调用 `/api/admin/**` 返回 403。
- [ ] Redis 未启动时核心业务仍可使用；启动后热门商品和详情可正常读取。
- [ ] JPG/JPEG/PNG 上传成功，并可通过 `/uploads/{filename}` 访问。

## 普通用户测试流程

1. 使用 `student1 / 123456` 登录。
2. 检查导航栏昵称和信用分。
3. 进入“发布闲置”，填写信息并上传图片。
4. 提交后进入“我的商品”，确认状态为 `PENDING`。
5. 编辑商品后确认重新进入待审核状态。
6. 对非已售商品执行下架，确认状态变为 `OFF_SHELF`。
7. 检查“我的订单”“我的收藏”“最近浏览”页面可正常打开。

## 管理员测试流程

1. 使用 `admin / 123456` 登录。
2. 进入 `/admin`，确认普通用户不会看到该入口。
3. 在商品审核列表通过或驳回待审核商品。
4. 在全部商品列表对在售商品执行违规下架。
5. 在用户管理中封禁普通用户，确认其无法继续通过 JWT 访问受保护接口。
6. 解封用户后确认可以重新登录。
7. 确认用户数、商品数、订单数和交易总金额指标显示正常。

## 商品交易完整流程

1. `student1` 发布商品。
2. `admin` 审核通过，商品状态变为 `ON_SALE`。
3. `student2` 打开商品详情并下单。
4. 在 `student2` 的“我购买的”中确认订单状态为 `CREATED`。
5. 在 `student1` 的“我出售的”中确认收到同一订单。
6. `student1` 点击确认完成。
7. 确认订单状态变为 `COMPLETED`，商品状态变为 `SOLD`。
8. 确认已售商品不能再次下单。

取消分支：

1. 买家对另一个在售商品下单。
2. 买家点击取消订单。
3. 确认状态变为 `CANCELLED`，卖家不能再确认完成。

## 收藏与浏览历史测试

1. 登录后打开一个商品详情，确认详情初始收藏状态正确。
2. 点击收藏，按钮变为“已收藏”。
3. 刷新详情页，确认收藏状态仍为已收藏。
4. 在“我的收藏”中确认商品存在。
5. 再次收藏同一商品，数据库不应出现重复记录。
6. 取消收藏，确认详情按钮和收藏列表同步更新。
7. 连续多次访问同一商品详情。
8. 在“最近浏览”中确认该商品只出现一次，并排在较新位置。
9. 访问另一个商品，确认最近浏览按最后访问时间倒序排列。

## 信用分变化测试

1. 完成一笔 `student2` 购买、`student1` 出售的订单。
2. 记录卖家当前信用分。
3. 买家提交 5 星评价，确认卖家信用分增加 2。
4. 确认同一订单不能重复评价。
5. 分别验证 4/3/2/1 星对应 `+1/0/-2/-5`。
6. 验证信用分不会低于 0，也不会高于 150。
7. 未完成订单不能评价，卖家不能评价自己的订单。

## 后台数据看板测试

1. 使用管理员进入 `/admin`。
2. 确认商品状态饼图、分类商品柱状图和 7 日订单折线图均显示。
3. 调整浏览器窗口大小，确认图表自动适应宽度。
4. 新增并审核商品，刷新页面后确认商品总数和图表数据变化。
5. 创建订单，确认订单总数及当天趋势变化。
6. 完成订单，确认交易总金额增加。
7. 检查接口：

```text
GET /api/admin/stat/overview
GET /api/admin/stat/goods-status
GET /api/admin/stat/category-goods
GET /api/admin/stat/order-trend
```

8. 使用普通用户 Token 请求以上接口，确认返回 403。

## 智能交易助手 Agent 测试

### 发布建议

1. 登录后进入 `/publish`。
2. 填写分类、原价、标题和描述。
3. 点击“智能生成发布建议”。
4. 确认返回优化标题、描述、建议价格、价格区间和建议列表。
5. 点击“一键应用建议”，确认标题、描述和售价写回表单。
6. 分别使用短描述和完整描述，确认 `canPublish` 与建议内容会变化。

### 价格评估

调用 `POST /api/agent/price-advice`，分别测试：

- 描述包含“全新未拆封”，区间应接近原价的 75%-90%。
- 描述包含“九成新”，区间应接近原价的 60%-75%。
- 描述包含“有划痕/维修”，区间应明显下降。
- 不填写原价时，应参考同分类在售均价。

### 风险分析

1. 登录后打开商品详情。
2. 确认显示风险等级、风险分数、原因和购买建议。
3. 验证无图片、短描述、低信用、异常低价会提高风险分。
4. 未登录时确认显示登录引导，接口直接访问返回 401。

### 个性化推荐

1. 使用新用户访问首页，确认无行为数据时推荐平台热门商品。
2. 浏览并收藏某分类商品。
3. 刷新首页，确认智能推荐优先出现相同分类商品。
4. 确认推荐结果不包含当前用户自己的商品和非 `ON_SALE` 商品。
5. 确认最多返回 8 条。

### 智能问答

进入 `/agent`，测试以下问题：

- “这个商品值得买吗”：填写商品 ID，应返回风险结论和同类商品。
- “这个价格合理吗”：填写商品 ID，应返回价格区间判断。
- “给我推荐一些数码产品”：应返回数码分类在售商品。
- “我的商品应该卖多少钱”：关联商品后应返回建议价格。
- “怎么提高信用分”：应返回评价规则与改进建议。
- 输入无法识别的问题：应返回能力说明和可点击问题建议。

### Agent 权限

- [ ] 未登录调用任意 `/api/agent/**` 返回 401。
- [ ] USER 和 ADMIN 登录后均可访问 Agent。
- [ ] Agent 不应修改商品、订单、评价等业务数据。

## 构建验收

后端：

```powershell
cd campus-trade-server
mvn clean package -DskipTests
```

前端：

```powershell
cd campus-trade-web
npm.cmd install
npm.cmd run build
```

- [ ] 后端输出 `BUILD SUCCESS`。
- [ ] 前端生成 `dist` 目录。
- [ ] Vite 仅有包体积警告时不影响运行。
